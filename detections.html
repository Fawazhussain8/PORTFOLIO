Detection Engineering

Due to confidentiality, detection rules are generalized representations of real-world implementations.

Privilege Group Change Monitoring
T1098 - Account Manipulation Windows Security High

Platform: Log360

title: Privileged Group Modification
logsource: Windows Security
detection:
  EventID: 4728, 4732, 4756
condition: selection

Detection Logic:
Alert on addition of users to privileged groups (Administrators, Domain Admins).

Why it matters:
Indicates privilege escalation or unauthorized access expansion.

My Contribution:

  • Filtered logs to focus on privilege-related events
  • Configured high-severity alert profile
  • Reduced noise from non-critical group changes
Brute Force Login Detection
T1110 - Brute Force Windows Security Medium
title: Multiple Failed Logins
logsource: Windows Security
detection:
  EventID: 4625
condition: threshold > 5 in short timeframe

Detection Logic:
Multiple failed login attempts within a short time window.

Enhancements:

  • Grouped by username + source IP
  • Implemented threshold-based alerting

Impact:
Early detection of password guessing attacks.
